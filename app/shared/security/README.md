# Security 模块说明

`app/shared/security/` 目前承载的是 Prompt 注入防御相关的轻量工具，而不是完整的安全中台。它的核心目标是给 Agent 节点建立最外层的用户输入信任边界。

## 结构分工

- `__init__.py`
  - 提供 `wrap_user_message()`。
  - 负责把原始用户输入转义后包进 `<user_message>` 标签，减少伪造闭合标签的风险。

## 当前边界

- `security/` 只处理轻量文本防护，不负责鉴权、权限控制或更复杂的安全编排。
- 它提供的是“进入 Prompt 前”的一层保护，真正的多层防御仍然要靠 `app/chat/` 里的路由、Guardrails 和 Cypher 写操作限制共同完成。

## 后续维护建议

- 如果只是新增 Prompt 级输入清洗或包裹规则，优先放在这里集中管理。
- 如果逻辑开始依赖具体业务状态、数据库或 HTTP 请求对象，说明它已经超出这个模块的边界，应上移到更合适的流程层。
